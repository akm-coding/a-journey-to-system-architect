**docker-guide.md** — Docker concepts study guide covering:
- What Docker solves (works on my machine problem, dependency isolation, reproducible deploys)
- Core concepts: images vs containers, Docker daemon, Docker Hub/registries
- Dockerfile anatomy: FROM, WORKDIR, COPY, RUN, EXPOSE, CMD — explain each instruction

**docker-exercise.md** — Hands-on exercise with the incremental approach:
1. Build a naive single-stage Dockerfile for the React app (FROM node:20-alpine, install all deps, copy everything, build, serve with a simple HTTP server or node). Time the build and check image size with `docker images`
2. Identify problems: huge image (node_modules + dev deps in prod), slow rebuilds on code changes
3. Refactor to multi-stage: Stage 1 (node:20-alpine AS build) installs deps and runs pnpm build:client, Stage 2 (nginx:alpine) copies only dist/client and nginx.conf
4. Compare: time the multi-stage build, check image size — concrete proof of improvement
5. Layer caching demo: change a source file, rebuild, observe which layers are cached vs rebuilt. Then show the bad pattern (COPY . . before install) vs good pattern (COPY package.json first)
6. Build the API Dockerfile (provided pre-written per user decision — exercise shows it and explains each line)
7. Verification: docker build both images, docker run the frontend on port 8080, curl localhost:8080 to confirm
- Include cleanup commands at end (docker rmi, docker system prune)

- Container commands (run, stop, rm, logs, exec, inspect)
- Image commands (images, rmi, tag, history, system prune)
- Dockerfile instructions table (FROM, WORKDIR, COPY, RUN, EXPOSE, CMD, USER, ARG, ENV)
- Multi-stage build pattern template
- Common troubleshooting commands table

**app/Dockerfile** — React frontend multi-stage build:
- Stage 1 (FROM node:20-alpine AS build): WORKDIR /app, COPY package.json pnpm-lock.yaml, RUN corepack enable and pnpm install --frozen-lockfile, COPY . ., RUN pnpm build:client
- Stage 2 (FROM nginx:alpine): COPY nginx.conf /etc/nginx/conf.d/default.conf, COPY --from=build /app/dist/client /usr/share/nginx/html, EXPOSE 80, CMD ["nginx", "-g", "daemon off;"]
- Add comments explaining each instruction for learning purposes

**app/Dockerfile.api** — Node API (provided pre-written per user decision):
- FROM node:20-alpine, WORKDIR /app
- RUN corepack enable
- COPY package.json pnpm-lock.yaml, RUN pnpm install --frozen-lockfile --prod
- COPY dist/server ./dist/server
- COPY src/server/db ./src/server/db (needed for Drizzle schema/migrations at runtime)
- Security: RUN addgroup -S appgroup && adduser -S appuser -G appgroup, USER appuser
- EXPOSE 3000, CMD ["node", "dist/server/index.js"]
- Add comment noting: requires `pnpm build:server` before docker build

**app/nginx.conf** — SPA config with API reverse proxy:
- listen 80, root /usr/share/nginx/html, index index.html
- gzip on with text/css, application/javascript, application/json types, min_length 256
- location /api/ { proxy_pass http://api:3000; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; }
- location / { try_files $uri $uri/ /index.html; } (SPA routing fallback)
- location /assets/ { expires 1y; add_header Cache-Control "public, immutable"; } (Vite hashed assets)
- Add comments explaining each directive (especially try_files for SPA routing and why /api/ proxies to the api service name in Docker Compose)

**app/.dockerignore**:
