Create three files in docs/phase-03/ following the concept-then-build pattern.

**ecr-guide.md** — ECR concepts:
- What ECR is: AWS-managed Docker registry, integrates with IAM for auth, integrates with ECS/Fargate (Phase 6)
- ECR vs Docker Hub: no rate limits with AWS auth, private by default, IAM-based access control
- Authentication flow: aws ecr get-login-password generates a temporary token (12-hour expiry), piped to docker login. Explain why this is needed (ECR is private, Docker CLI needs credentials)
- Repository structure: one repo per image (e.g., ecommerce-frontend, ecommerce-api), each repo holds multiple tagged versions
- Tagging strategy: git commit SHA for traceability (know exactly which code is in which image), "latest" for convenience (always points to newest). Explain why relying on "latest" alone is dangerous in production
- Lifecycle policies: automate image cleanup to control costs. Rules for: expire untagged images after N days, keep only last N tagged images. JSON policy format with rulePriority, selection, action
- Cost awareness: ECR charges for storage. Without lifecycle policies, images accumulate indefinitely
- Use WHY/GOTCHA callouts, including the 12-hour auth expiry pitfall

**ecr-exercise.md** — Hands-on exercise:
1. Set variables: AWS_ACCOUNT_ID (from aws sts get-caller-identity), AWS_REGION, REPO_NAME
2. Create ECR repository: aws ecr create-repository --repository-name ecommerce-frontend
3. Authenticate: aws ecr get-login-password | docker login
4. Build the frontend image: docker build -t ecommerce-frontend . (from app/ directory)
5. Tag with commit SHA: docker tag ecommerce-frontend $ACCOUNT.dkr.ecr.$REGION.amazonaws.com/ecommerce-frontend:$(git rev-parse --short HEAD)
6. Tag with latest: docker tag ecommerce-frontend $ACCOUNT.dkr.ecr.$REGION.amazonaws.com/ecommerce-frontend:latest
7. Push both tags: docker push (both)
8. Verify in console: aws ecr describe-images --repository-name ecommerce-frontend (also mention checking AWS Console for visual verification)
9. Repeat for ecommerce-api (create repo, build with Dockerfile.api, tag, push)
10. Apply lifecycle policy: aws ecr put-lifecycle-policy with the JSON from research (keep last 10, expire untagged after 7 days)
11. Verify policy: aws ecr get-lifecycle-policy
12. Cleanup: reference the teardown script
- Include troubleshooting: auth token expired, "no basic auth credentials", wrong region

- Tag and push workflow (copy-pasteable with variable placeholders)
- Lifecycle policy commands (put, get, delete)
