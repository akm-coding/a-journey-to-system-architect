  <action>
    Update app/package.json to add all dependencies per the research standard stack:
    - Runtime: react, react-dom, express, pg, drizzle-orm, dotenv, cors
    - Dev: typescript, @types/react, @types/react-dom, @types/express, @types/pg, @types/cors, drizzle-kit, tsx, vite, @vitejs/plugin-react, react-router-dom (for client routing)
    - Scripts: "dev:server" (tsx watch src/server/index.ts), "dev:client" (vite --config src/client/vite.config.ts), "build:server" (tsc for server files), "build:client" (vite build --config src/client/vite.config.ts), "seed" (tsx src/server/db/seed.ts), "db:push" (drizzle-kit push), "db:generate" (drizzle-kit generate)

    Create app/tsconfig.json with:
    - target: ES2022, module: NodeNext, moduleResolution: NodeNext
    - strict: true, esModuleInterop: true
    - outDir: ./dist, rootDir: ./src
    - Include both server and client source directories

    Create app/src/server/db/schema.ts with Drizzle table definitions per the research schema:
    - products: id (identity), name (varchar 255), description (text), price (numeric 10,2), imageUrl (varchar 500)
    - orders: id (identity), status (varchar 50, default "pending"), totalAmount (numeric 10,2), createdAt (timestamp, defaultNow)
    - orderItems: id (identity), orderId (FK to orders), productId (FK to products), quantity (integer), price (numeric 10,2)

    Create app/src/server/db/index.ts: Initialize Drizzle client with process.env.DATABASE_URL. Export the db instance.

    Create app/src/server/db/seed.ts: Insert 6-8 sample products (electronics/general store items with realistic names, descriptions, prices). Use Drizzle insert. Log what was inserted. Exit after seeding.

    Create app/src/server/routes/products.ts: Express Router with:
    - GET /api/products -- return all products
    - GET /api/products/:id -- return single product by id, 404 if not found

    Create app/src/server/routes/orders.ts: Express Router with:
    - POST /api/orders -- accept { items: [{productId, quantity}] }, create order + order_items, return order with 201
    - GET /api/orders/:id -- return order with its items

    Create app/src/server/routes/cart.ts: Express Router with:
    - This is a pass-through -- cart is client-side only (localStorage). Include a GET /api/cart/info endpoint that returns a message explaining cart is stored client-side. This is intentional per research: no auth = no server-side cart.

    Create app/src/server/index.ts: Express app with:
    - JSON body parsing (express.json())
    - CORS middleware (for local dev when client/server on different ports)
    - GET /api/health returning { status: "ok", timestamp }
    - Mount product, order, and cart routes
    - Listen on process.env.PORT || 3000

    Create app/drizzle.config.ts per research: schema path, postgresql dialect, DATABASE_URL from env.

    Create app/ecosystem.config.js per research PM2 pattern: name "ecommerce-api", script "./dist/server/index.js", fork mode, 1 instance, PORT 3000, NODE_ENV production. Include max_restarts: 10, restart_delay: 1000.

    Run `pnpm install` from the app directory after updating package.json.
