Create the reusable VPC + Security Groups module per CONTEXT.md decision ("VPC + Security Groups module as the reusable module").

**infra/modules/vpc/variables.tf:**
- `vpc_cidr` (string, default "10.0.0.0/16", validation with `can(cidrhost(...))`)
- `environment` (string, validation: must be "staging" or "production")
- `public_subnet_cidrs` (list(string), description)
- `private_subnet_cidrs` (list(string), description)
- `availability_zones` (list(string), description)

Per CONTEXT.md: all variables have type constraints, validation blocks, and descriptions.

**infra/modules/vpc/main.tf:**
All resources annotated with comment blocks explaining what and why (same as Phase 3 annotated Dockerfiles).

Resources to create:
1. `aws_vpc` -- CIDR from variable, enable_dns_hostnames = true, enable_dns_support = true
2. `aws_internet_gateway` -- attached to VPC
3. `aws_subnet` (public) -- use `count` or `for_each` over `var.public_subnet_cidrs`, map_public_ip_on_launch = true, each in different AZ
4. `aws_subnet` (private) -- use `count` or `for_each` over `var.private_subnet_cidrs`, each in different AZ (needed for RDS subnet group -- RESEARCH.md pitfall #6)
5. `aws_route_table` (public) -- route 0.0.0.0/0 to IGW
6. `aws_route_table_association` (public) -- associate each public subnet
7. `aws_route_table` (private) -- no internet route (RDS doesn't need outbound)

Security groups (use separate `aws_security_group_rule` resources to avoid circular deps -- per RESEARCH.md pitfall #4):
8. `aws_security_group` "app" -- for EC2 instances
   - Ingress: port 80 (HTTP), port 443 (HTTPS), port 22 (SSH) from 0.0.0.0/0
   - Egress: all traffic
9. `aws_security_group` "rds" -- for RDS instances
   - Ingress: port 5432 from app SG only (SG referencing pattern from Phase 2)
   - Egress: all traffic
10. `aws_security_group` "rds_proxy" -- for RDS Proxy
    - Ingress: port 5432 from app SG
    - Egress: port 5432 to rds SG

Tag all resources with `Name = "${var.environment}-{resource}"` and `Environment = var.environment`.

**infra/modules/vpc/outputs.tf:**
- `vpc_id`
- `public_subnet_ids` (list)
- `private_subnet_ids` (list)
- `app_sg_id`
- `rds_sg_id`
- `rds_proxy_sg_id`
All with descriptions.
