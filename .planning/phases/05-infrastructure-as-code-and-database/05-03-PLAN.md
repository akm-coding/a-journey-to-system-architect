Create `docs/phase-05/02-database-patterns/guide.md` covering production database patterns. Follow the concept-then-build pattern (WHY/WHAT).

Sections:
1. **Database Migrations in Production** -- Why migrations matter (schema drift, reproducibility). The drizzle-kit push workflow: SSH into EC2, set DATABASE_URL to RDS direct endpoint (not proxy -- per RESEARCH.md pitfall #7 note: migrations use direct endpoint), run `npx drizzle-kit push`. Explain idempotency. Show the pre-deploy migration step in the deployment workflow. Per CONTEXT.md: "SSH in, run drizzle-kit push (same as Phase 4 pattern, now documented in runbook)"

2. **Connection Pooling with RDS Proxy** -- Why connection pooling matters (connection limits, Lambda cold starts, multi-instance scaling). How RDS Proxy works: sits between app and RDS, manages a pool of database connections. The app connects to the proxy endpoint (not direct RDS). Proxy requires: Secrets Manager for credentials, IAM role, VPC placement. Cost consideration (~$22/month for learning). How to verify pooling works: check RDS connection count vs app instance count

3. **Automated Backups** -- RDS automated daily backups: how they work, backup window, retention period (7 days per decision). Point-in-time recovery: what it is, how RDS does it (transaction logs), restoration creates a NEW instance. Backup storage costs (free up to 1x DB size)

4. **Manual Snapshots** -- When to take them (before major changes, before destroy). How to create via CLI (`aws rds create-db-snapshot`). Difference from automated backups (persist until deleted, not subject to retention period)

5. **Multi-AZ vs Single-AZ** -- Per CONTEXT.md: "Study both patterns; deploy single-AZ to keep costs low." Explain Multi-AZ: synchronous standby replica in different AZ, automatic failover. When to use it (production with uptime SLAs). Cost impact (~2x). Show the Terraform config difference (just `multi_az = true`)

6. **Security Patterns** -- Private subnet placement (no public access), SG referencing (only app SG can reach RDS), encrypted storage, Secrets Manager for credentials (not plaintext in tfvars)

Include WHY and GOTCHA callouts throughout.

**docs/phase-05/03-exercises/exercise.md** -- Progressive hands-on exercises following the Phase 4 pattern (objective, why-it-matters, steps, expected output, what-you-learned). Per CONTEXT.md: "Destroy and rebuild exercise as final exercise (proves IaC reproducibility)."

Exercise 1: Bootstrap the State Backend
- cd infra/bootstrap, terraform init, terraform plan (review output), terraform apply
- Verify: S3 bucket exists, DynamoDB table exists, ECR repo exists
- Record output values (bucket name, table name) -- needed for environment configs

Exercise 2: Deploy Staging Infrastructure
- cd infra/staging, set TF_VAR_db_username and TF_VAR_db_password as env vars
- terraform init (note: downloads providers, configures S3 backend)
- terraform plan (review the full resource list, understand what will be created)
- terraform apply (watch creation order, note dependency resolution)
- Verify: VPC in console, EC2 running, RDS available, RDS Proxy available

Exercise 3: Explore Terraform State
- terraform state list (see all managed resources)
- terraform state show aws_db_instance.main (see resource details)
- terraform output (see all outputs, note proxy endpoint)
- Discuss: why you should never edit state manually

Exercise 4: Run Database Migration
- SSH into EC2 instance using terraform output ec2_public_ip
- Set DATABASE_URL to direct RDS endpoint (from terraform output rds_direct_endpoint)
- Run npx drizzle-kit push
- Verify: tables created, connect via proxy endpoint from app

Exercise 5: Create Manual Snapshot and Test Backup
- aws rds create-db-snapshot (manual snapshot before changes)
- Verify snapshot in console/CLI
- Check automated backup in console (should show daily backup)
- Discuss: point-in-time recovery options

Exercise 6: Destroy and Rebuild (the capstone exercise)
- terraform destroy in staging (watch dependency-ordered teardown)
- Verify: all resources gone
- terraform apply again (prove reproducibility)
- Verify: everything works again, same as before
- Per CONTEXT.md: "Destroy and rebuild exercise as final exercise (proves IaC reproducibility, matches success criteria #5)"
