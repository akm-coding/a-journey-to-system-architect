Create `docs/phase-04/exercises.md` -- hands-on exercises that use the workflow from Plan 01. Per user decisions, include these specific exercises:

**Exercise 1: Set Up GitHub Environments and Secrets**
- Walk through creating `staging` and `production` environments in GitHub repo settings
- Add required secrets per environment: EC2_HOST, EC2_USERNAME, SSH_PRIVATE_KEY, AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY
- Add environment variables: AWS_REGION, APP_URL
- Set branch restrictions (production: main only, staging: develop only)
- Verification: screenshot or list of configured environments

**Exercise 2: Trigger Your First Pipeline Run**
- Create the `develop` branch from main
- Push a small change to develop (e.g., edit a comment)
- Watch the workflow run in GitHub Actions tab
- Walk through each job's logs, explain what to look for
- Verify staging deployment via the staging EC2 URL

**Exercise 3: Break a Test and Watch the Pipeline Fail**
- Per user decision: "intentionally break a test and watch pipeline fail"
- Edit a test to assert something wrong
- Push to develop, observe lint-and-test job fails
- Observe build-images and push-and-deploy are SKIPPED (not failed)
- Fix the test, push again, observe green pipeline
- Key learning: the pipeline catches failures before they reach production

**Exercise 4: Add a New Secret**
- Per user decision: "add a new secret"
- Add a new environment variable (e.g., NODE_ENV) to GitHub Environments
- Reference it in the workflow or compose environment
- Deploy and verify it is available on the EC2 instance

**Exercise 5: Modify the Deploy Step**
- Per user decision: "modify deploy step"
- Add a step to the SSH deploy that prints the currently running container versions after deploy
- Push change, observe the updated deploy output in GitHub Actions logs

**Exercise 6: Promote to Production**
- Merge develop into main via PR
- Observe the pipeline runs against the production environment
- Verify the production EC2 has the updated app
- Key learning: same pipeline, different environment

Each exercise should have: objective, prerequisites, step-by-step instructions, expected output, and "what you learned" summary. Progressive difficulty (per Claude's discretion).

For each pitfall: **Symptom** (what you see), **Cause** (why it happens), **Fix** (how to resolve), **Prevention** (how to avoid in the future). Use a consistent format throughout.
