# Phase 1: Rebuild Log

## The Rebuild Challenge

After completing each guided exercise, your challenge is to **tear down everything** and **rebuild it from scratch without looking at the instructions**. This is how you internalize the knowledge instead of just following steps.

### How It Works

1. Complete the guided exercise for a topic
2. Tear down the resources you created (use `scripts/teardown-checklist.sh` as a reminder)
3. Set a timer
4. Rebuild everything from memory
5. Log your attempt below

### What to Track

- **Attempt #**: Number your attempts (you may need 2-3 per topic)
- **Time Taken**: How long the rebuild took
- **Issues Encountered**: Where you got stuck or had to look something up
- **Resolved?**: Did you finish the rebuild successfully?

The goal is to rebuild faster and with fewer lookups each time.

## Benchmark Targets

These are rough targets for a successful rebuild (not including setup/teardown time):

| Topic | Target Time | Notes |
|-------|-------------|-------|
| AWS Account Setup | 15 min | Mostly console clicks, fast once familiar |
| Linux Fundamentals | 20 min | SSH + navigate + edit files |
| Processes & Permissions | 20 min | Services + permissions + env vars |
| Networking & VPC | 30 min | VPC + subnets + route tables + IGW |
| Security Groups | 15 min | Create + configure rules + verify |
| DNS | 20 min | Create records + verify propagation |
| SSL/TLS | 25 min | Certbot + Nginx config + verify HTTPS |
